## 1. 阶段定位

M01 是整个项目的**硬前置阶段**，所有后续阶段（M02–M05）都强依赖本阶段产出的训练 / 推理 / 评测运行环境。

本阶段不涉及任何业务逻辑实现，只解决三件事：

1. **硬件层**：验证 2×A100-80G 训练节点可用，NVLink 互联正常。
2. **软件层**：构建分层 Docker 镜像（base + 4 种分布式后端 layer + 推理 layer + 评测 layer）。
3. **烟雾层**：在四个核心组件（LLaMA-Factory SFT、DeepSpeed ZeRO3、vLLM、xinference）上各跑一次最小冒烟用例，确认链路通畅。

## 3. 核心任务

### 3.1 Docker 基础镜像构建

采用**基础镜像 + 后端 layer**的分层设计，避免单个超大镜像：

```
# 基础镜像
copaw-dpo-base:latest
  - python3.10 + cuda12.4 + cudnn8.9 + pytorch2.4 + transformers + LLaMA-Factory

# 后端 layer（在 base 之上叠加）
copaw-dpo-deepspeed:latest   # + deepspeed + nvtx
copaw-dpo-fsdp:latest        # + pytorch fsdp + nvtx
copaw-dpo-megatron:latest    # + megatron-lm + apex
copaw-dpo-accelerate:latest  # + accelerate

# 推理 layer
copaw-dpo-infer-vllm:latest   # + vllm
copaw-dpo-infer-xinfer:latest # + xinference + docker-compose

# 评测 layer
copaw-dpo-eval:latest         # + sacrebleu + rouge-score + judge 模型
```

构建脚本示例：

```bash
# base 镜像
docker build -f deploy/Dockerfile.base -t copaw-dpo-base:latest .

# 4 种分布式后端 layer
for backend in deepspeed fsdp megatron accelerate; do
    docker build -f deploy/Dockerfile.$backend -t copaw-dpo-$backend:latest .
done

# 推理 layer
docker build -f deploy/Dockerfile.infer-vllm    -t copaw-dpo-infer-vllm:latest .
docker build -f deploy/Dockerfile.infer-xinfer  -t copaw-dpo-infer-xinfer:latest .

# 评测 layer
docker build -f deploy/Dockerfile.eval -t copaw-dpo-eval:latest .
```

### 3.3 LLaMA-Factory 单卡 SFT 冒烟

用极小的虚构数据集（≤ 50 条）跑一次 SFT，确认：

- yaml 配置解析正确
- 模型加载成功（Qwen2.5-1.5B-Instruct）
- LoRA 注入成功
- 1 epoch 后 loss 有下降趋势

最小 yaml 示例：

```yaml
# configs/smoke_lora_qwen2_5_1_5b.yaml
model_name_or_path: Qwen/Qwen2.5-1.5B-Instruct
stage: sft
do_train: true
finetuning_type: lora
lora_target: q_proj,v_proj
lora_rank: 8
dataset_dir: data/smoke
dataset: smoke_sft
template: qwen
output_dir: saves/smoke_lora
per_device_train_batch_size: 2
num_train_epochs: 1
bf16: true
logging_steps: 1
save_steps: 100
report_to: tensorboard
```

### 3.4 vLLM 单卡推理冒烟

```python
# scripts/smoke_vllm.py
from vllm import LLM, SamplingParams

llm = LLM(model="Qwen/Qwen2.5-1.5B-Instruct",
          tensor_parallel_size=1,
          gpu_memory_utilization=0.85,
          max_model_len=2048)
params = SamplingParams(temperature=0.7, max_tokens=64)
out = llm.generate(["保险等待期内确诊是否赔付？"], params)
print(out[0].outputs[0].text)
```

## 7. 详细技术规范

### 7.1 训练节点硬件要求

| 组件 | 规格 | 数量 | 备注 |
|------|------|------|------|
| GPU | NVIDIA A100-80G | 2 | NVLink 互联（首选）或 PCIe 4.0 |
| CPU | 32 核以上（Intel Xeon 或 AMD EPYC） | 1 | 数据加载需要 |
| 内存 | ≥ 256 GB DDR4 | - | 数据缓存 |
| 存储 | NVMe SSD ≥ 2 TB | - | 本地数据集 + checkpoint |
| 网络 | 10 GbE × 2（bond） | - | 与司内存储 / RAG 端通信 |
| IB（可选） | InfiniBand HDR 200 Gbps | - | Megatron TP 时启用 |

### 7.2 软件依赖版本

| 依赖 | 版本 | 备注 |
|------|------|------|
| Python | 3.10 | LLaMA-Factory 要求 3.10+ |
| PyTorch | 2.4.0+ | 支持 FSDP v2 |
| CUDA | 12.4 | 与 PyTorch 2.4 配套 |
| NCCL | 2.21+ | 分布式通信 |
| transformers | 4.43+ | LLaMA-Factory 配套 |
| LLaMA-Factory | 0.9.x | 训练流水线编排 |
| DeepSpeed | 0.14+ | ZeRO3 主路径 |
| accelerate | 0.33+ | accelerate 备选 |
| vLLM | 0.5+ | 推理加速 |
| xinference | 0.15+ | 推理备选 |
| sacrebleu | 2.4+ | BLEU |
| rouge-score | 0.1.2+ | ROUGE |

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| NVLink 不可用（仅 PCIe 互联） | 跨卡通信慢，DeepSpeed/FSDP 性能下降 | 接受 PCIe 性能；调整 gradient_accumulation 补偿 |
| CUDA / PyTorch 版本不匹配 | 镜像无法构建或运行 | 严格按版本矩阵锁版本（CUDA 12.4 + PyTorch 2.4） |
| xinference 国产 GPU 驱动问题 | 推理节点选型受限 | 优先选 A100-80G；xinference 仅作备选 |
| Docker 镜像过大（> 20 GB） | CI/CD 推送慢 | 分层设计，仅 base 镜像含 CUDA，其他 layer 按需拉取 |
| LLaMA-Factory 升级导致 yaml 不兼容 | 训练流水线失效 | 锁版本 `LLaMA-Factory==0.9.x`，CI 烟雾测试守护 |
