## 3. 核心任务

### 3.1 推理后端抽象层（M-INFER）

**`InferBackend` 抽象类**：

```python
# m_infer/base.py
from abc import ABC, abstractmethod
from dataclasses import dataclass
from typing import Optional

@dataclass
class InferRequest:
    prompt: str
    max_new_tokens: int = 512
    temperature: float = 0.7
    top_p: float = 0.9
    stop: Optional[list[str]] = None
    request_id: Optional[str] = None

@dataclass
class InferResponse:
    text: str
    prompt_tokens: int
    generated_tokens: int
    latency_ms: float        # 首 token 时延
    total_latency_ms: float  # 全句时延
    request_id: Optional[str] = None


class InferBackend(ABC):
    """推理后端抽象类"""

    @abstractmethod
    def load(self, model_path: str, **kwargs) -> None: ...

    @abstractmethod
    def infer(self, req: InferRequest) -> InferResponse: ...

    @abstractmethod
    def batch_infer(self, reqs: list[InferRequest]) -> list[InferResponse]: ...

    @abstractmethod
    def shutdown(self) -> None: ...


class InferBackendFactory:
    _registry = {
        "vllm":      "m_infer.vllm_backend:VLLMBackend",
        "xinference": "m_infer.xinference_backend:XinferenceBackend",
    }

    @classmethod
    def build(cls, name: str, model_path: str, **kwargs) -> InferBackend:
        if name not in cls._registry:
            raise ValueError(f"unsupported infer backend: {name}")
        module_path, cls_name = cls._registry[name].split(":")
        backend_cls = importlib.import_module(module_path).__dict__[cls_name]
        backend = backend_cls()
        backend.load(model_path, **kwargs)
        return backend
```

### 3.2 vLLM 后端实现（FR-08 默认）

```python
# m_infer/vllm_backend.py
from vllm import LLM, SamplingParams
import time

class VLLMBackend(InferBackend):
    def load(self, model_path: str, tensor_parallel_size: int = 1, **kw):
        self._llm = LLM(
            model=model_path,
            tensor_parallel_size=tensor_parallel_size,
            dtype="bfloat16",
            gpu_memory_utilization=0.85,
            max_model_len=2048,
            enforce_eager=False,    # 启用 cuda graph
        )

    def infer(self, req: InferRequest) -> InferResponse:
        params = SamplingParams(
            temperature=req.temperature,
            top_p=req.top_p,
            max_tokens=req.max_new_tokens,
            stop=req.stop,
        )
        t0 = time.perf_counter()
        outputs = self._llm.generate([req.prompt], params, use_tqdm=False)
        total_ms = (time.perf_counter() - t0) * 1000
        out = outputs[0]
        return InferResponse(
            text=out.outputs[0].text,
            prompt_tokens=len(out.prompt_token_ids),
            generated_tokens=len(out.outputs[0].token_ids),
            latency_ms=out.metrics.first_token_time if out.metrics else 0.0,
            total_latency_ms=total_ms,
            request_id=req.request_id,
        )
```

**vLLM 关键技术**：

- **PagedAttention**：分页式 KV cache 管理，显存利用率提升 4× 以上。
- **Continuous batching**：请求级调度，不等整批完成即可插入新请求。
- **Chunked prefill**：长 prompt 分块 prefill，避免阻塞短请求 decode。

### 3.3 xinference 后端实现（FR-08 备选）

```python
# m_infer/xinference_backend.py
import requests
import time

class XinferenceBackend(InferBackend):
    """xinference 通过 HTTP / gRPC 与本地/远程 xinference 服务通信"""

    def load(self, model_path: str, server_endpoint: str = "http://127.0.0.1:9997",
             model_uid: Optional[str] = None, **kw):
        if model_uid is None:
            resp = requests.post(
                f"{server_endpoint}/v1/models",
                json={
                    "model_name": "qwen2_5_insurance",
                    "model_path": model_path,
                    "model_type": "LLM",
                },
            )
            model_uid = resp.json()["model_uid"]
        self._endpoint = server_endpoint
        self._model_uid = model_uid

    def infer(self, req: InferRequest) -> InferResponse:
        t0 = time.perf_counter()
        resp = requests.post(
            f"{self._endpoint}/v1/completions",
            json={
                "model": self._model_uid,
                "prompt": req.prompt,
                "max_tokens": req.max_new_tokens,
                "temperature": req.temperature,
                "top_p": req.top_p,
                "stop": req.stop,
            },
        )
        total_ms = (time.perf_counter() - t0) * 1000
        data = resp.json()
        return InferResponse(
            text=data["choices"][0]["text"],
            prompt_tokens=data["usage"]["prompt_tokens"],
            generated_tokens=data["usage"]["completion_tokens"],
            latency_ms=data.get("first_token_latency_ms", 0.0),
            total_latency_ms=total_ms,
            request_id=req.request_id,
        )
```

**xinference 关键技术**：

- 统一模型注册中心（xgboost / pytorch / LLM / embedding 一站式）。
- 内置 vLLM / transformers / llama.cpp 等多种推理后端，可按需选。
- 支持模型量化（GPTQ / AWQ / BNB），对国产 GPU 适配较好。

### 3.6 评测指标实现

| 指标 | 计算方式 | 适用场景 |
|------|----------|----------|
| Accuracy | 分类式问答（单选/多选/是/否）使用正则抽取 + 严格匹配；开放式问答使用 LLM-as-Judge 判定 0/1 | 医疗问答测试集中的客观题 |
| BLEU-4 | `sacrebleu.corpus_bleu(hyps, [refs])`，n-gram=4 + brevity penalty | 答案与参考答案的 n-gram 重合度 |
| ROUGE-L | `rouge_score.RougeScorer(['rougeL'])`，基于 LCS | 答案与参考答案的最长公共子序列重合度 |
| 推理时间 | `time.perf_counter()` 记录每条样本首 token 时延与全句时延 | 性能验收 |

**评测模块代码骨架**：

```python
# m_eval/metrics.py
import sacrebleu
from rouge_score import rouge_scorer

def accuracy_score(preds: list[str], refs: list[str]) -> float:
    """分类式问答：正则抽取 + 严格匹配；开放式：LLM-as-Judge"""
    correct = 0
    for p, r in zip(preds, refs):
        if normalize_answer(p) == normalize_answer(r):
            correct += 1
    return correct / len(preds) if preds else 0.0

def bleu_4_score(preds: list[str], refs: list[str]) -> float:
    """BLEU-4"""
    return sacrebleu.corpus_bleu(preds, [refs]).score / 100.0

def rouge_l_score(preds: list[str], refs: list[str]) -> float:
    """ROUGE-L (取所有样本均值)"""
    scorer = rouge_scorer.RougeScorer(["rougeL"], use_stemmer=True)
    scores = [scorer.score(r, p)["rougeL"].fmeasure for p, r in zip(preds, refs)]
    return sum(scores) / len(scores) if scores else 0.0
```

### 3.7 推理时间统计

```python
# m_eval/latency.py
import time
import numpy as np
from dataclasses import dataclass

@dataclass
class LatencyStat:
    p50_first_token_ms: float
    p95_first_token_ms: float
    p99_first_token_ms: float
    p50_total_ms: float
    p95_total_ms: float
    p99_total_ms: float
    throughput_samples_per_s: float

def aggregate_latency(records: list[InferResponse]) -> LatencyStat:
    first = np.array([r.latency_ms for r in records])
    total = np.array([r.total_latency_ms for r in records])
    elapsed = sum(r.total_latency_ms for r in records) / 1000.0
    return LatencyStat(
        p50_first_token_ms=float(np.percentile(first, 50)),
        p95_first_token_ms=float(np.percentile(first, 95)),
        p99_first_token_ms=float(np.percentile(first, 99)),
        p50_total_ms=float(np.percentile(total, 50)),
        p95_total_ms=float(np.percentile(total, 95)),
        p99_total_ms=float(np.percentile(total, 99)),
        throughput_samples_per_s=len(records) / elapsed,
    )
```

| vLLM 模式 | 1×A100-80G（推荐）或 1×A30-24G | PagedAttention 显存利用率高 |
| xinference 模式 | 1×A100-80G / 国产 GPU / CPU（量化后） | 灵活 |

### 7.2 评测工具选型

- **Accuracy**：分类式问答（单选/多选）使用正则抽取选项 + 字符串比对；开放式问答使用 LLM-as-Judge 作为辅助。
- **BLEU**：统一使用 `sacrebleu`（与 3.6 `m_eval/metrics.py` 的实现保持一致），输出 BLEU-4（tokenized 13a 模式 + 简洁惩罚）；不再混用 `nltk.translate.bleu_score`，避免与司内既有脚本的口径差异。
- **ROUGE**：使用 `rouge-score`（Google Research 官方实现）计算 ROUGE-1/2/L。
- **推理时间**：使用 `time.perf_counter` 在评测脚本中记录每条样本的首 token 时延与全句时延；统计 p50 / p95 / p99 与总吞吐量。

### 7.6 与司内普通 RAG 端对接的接口契约（详细版）

**端到端连通性测试脚本**：

```python
# demos/rag_connectivity/test_e2e.py
import requests

RAG_ENDPOINT = "http://127.0.0.1:8080/v1/insurance/qa"

test_queries = [
    "重疾险等待期内确诊是否赔付？",
    "百万医疗险的免赔额是怎么计算的？",
    "投保前未告知高血压，理赔会被拒吗？",
    # ... 共 10 条
]

def test_connectivity():
    success = 0
    for q in test_queries:
        resp = requests.post(RAG_ENDPOINT, json={
            "user_query": q,
            "context_docs": [],
            "max_new_tokens": 256,
            "temperature": 0.3,
        })
        if resp.status_code == 200 and resp.json().get("answer"):
            success += 1
            print(f"[OK] {q[:30]}... → {resp.json()['answer'][:60]}...")
        else:
            print(f"[FAIL] {q[:30]}... → status={resp.status_code}")
    print(f"\n连通成功率：{success}/{len(test_queries)}")
    assert success == len(test_queries), "连通性测试失败"

if __name__ == "__main__":
    test_connectivity()
```

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| vLLM 与 xinference API 差异 | 切换后报错 | 抽象层 `InferBackend` 屏蔽 |
| 司内 RAG 端接口变更 | 对接失败 | 适配层封装 `m_infer/rag_handler.py`，变更只改一处 |
| BLEU / ROUGE 计算与司内既有脚本不一致 | 评测口径争议 | 锁版本 + 文档明确公式 |
| LLM-as-Judge 输出不稳定 | accuracy 波动 | 多次采样取均值 / 固定温度 0.0 |
| xinference 国产 GPU 驱动问题 | 推理节点选型受限 | 优先选 A100-80G；xinference 仅作备选 |
| 评测耗时 > 30 min | 验收超时 | 启用 vLLM continuous batching；并发评测 |
| 推理延迟 p99 > 2.5 s | 性能不达标 | 调整 batch / 启用 cuda graph / 减少 max_new_tokens |
| RAG 端 `/v1/insurance/qa` 路由冲突 | 流量冲突 | 端口隔离 + URL 路径前缀 |
