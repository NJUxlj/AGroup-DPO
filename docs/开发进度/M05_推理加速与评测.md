## 6. 关键技术决策

### 6.1 vLLM 0.22.1 flashinfer 依赖问题

vLLM 0.22.1 的 V1 engine 默认依赖 `flashinfer` 采样器，但该包无法通过 pip 安装。
通过在 `VLLMBackend.load()` 中设置 `os.environ.setdefault("VLLM_USE_FLASHINFER_SAMPLER", "0")`
强制使用 PyTorch 原生采样器解决。vLLM 内部日志确认：
```
FlashInfer top-p/top-k sampling disabled via VLLM_USE_FLASHINFER_SAMPLER=0; using PyTorch-native sampler.
```

### 6.2 BLEU-4 空样本处理

`sacrebleu` 在 2 以下 token 的短文本中返回 0.0（需要 4-gram 无法计算），
这是标准行为。评测时应使用足够长的参考答案（≥10 tokens）。

### 6.3 ROUGE-L 中文支持

`rouge-score` 默认 tokenizer 不支持中文字符（按空格分词导致中文全为单字符）。
实际评测使用英文参考答案绕过，生产环境可考虑使用 `jieba` 分词预处理。

## 7. Baseline 对比报告 (D-M05-16)

### 7.1 评测结果

| 指标 | Baseline (no DPO) | DPO | 相对变化 |
|------|-------------------|-----|----------|
| Accuracy | 0.0000 | 0.0000 | N/A |
| BLEU-4 | 0.0000 | 0.0000 | N/A |
| ROUGE-L | 0.0686 | 0.0235 | **-65.7%** |
| p50 延迟 | 353ms | 261ms | **-25.9%** |
| 吞吐 | 2.8 samples/s | 3.8 samples/s | +35.7% |

### 7.2 分析

- **Accuracy/BLEU-4 为 0**：1.5B 小模型开放式问答与参考答案措辞差异大
- **ROUGE-L DPO 低于 baseline**：DPO 对齐后回答更简短聚焦，LCS 重叠减少
- **延迟 DPO 优于 baseline（-25.9%）**：DPO 输出更短，推理耗时更少
- **吞吐 DPO 更高（+35.7%）**：短输出带来更高吞吐

> 基于 10 条样本的趋势参考，全量对比需 1000+ 条评测集 + LLM-as-Judge。
