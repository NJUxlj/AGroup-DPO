## 4. 已知缺口

1. **Docker 镜像构建**：server2 无 Docker，暂以 conda 环境替代
2. **NCCL all-reduce**：RTX 5090 sm_120 的 NCCL 2.26.2 kernel 兼容性问题，barrier 通过即可确认通信链路，完整修复需 NCCL 2.27+

### 已解决缺口
- ✅ **xinference**：xinference 2.10.0 已安装在 llm 环境，通过 `enable_virtual_env=False` 参数成功启动并推理 5/5 通过。需注意 xinference 2.x API 与旧版不同，需使用 Python Client（`xinference.client.Client`）替代 curl 调用。
- ✅ **vLLM**：vLLM 0.22.1 已安装在独立 conda 环境（`vllm`）。解决路径：
  1. 从 0.7.3 → 0.8.5 → 0.22.1 多次升级以匹配 torch 2.11.0+cu130（支持 sm_120）
  2. flashinfer 0.6.11 因 `_compat` 名称冲突无法加载，卸载 flashinfer 后设 `VLLM_USE_FLASHINFER_SAMPLER=0` 使用 PyTorch 原生采样
  3. 模型加载 ~99s，推理 5 条 ~40s，回答质量良好
